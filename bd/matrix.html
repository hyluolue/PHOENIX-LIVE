6.5 🔥 凤凰产品开发逻辑 = 人群 × 场景 × 品类 三维交集

核心原则：差异化 ≠ 参数堆叠 · 差异化 = 找到"人群 × 场景 × 品类"三维交集的空白位置
❌ 错误做法（参数差异化）：
"比 Veloce 轻 0.5kg" / "比 Core 多 3 档变速" → 同质化 = 价格战
✅ 正确做法（人群 × 场景 × 品类）：
"给越野爱好者在周末山地场景用的27.5 寸 MTB，加 UCI 情绪 + 夜光烤漆 + 静电涂装 = 真实差异化"
🎯 凤凰产品决策树（V2.3 战略 → V2.9.3 升级）
  1. 人群定位：先选目标人群 (越野爱好者 + 比赛玩家) → 不要先想"我要做什么车"
  2. 场景匹配：明确该人群的典型使用场景 (周末山地 / 长途比赛) → 决定 27.5/29 寸 / UCI 公路
  3. 品类聚焦：匹配品类 (MTB / 公路) → 跳过 Hybrid / 折叠 / 童车 永久不参与
  4. 功能价值：在选定品类里选 中高端价段 (15-40K MTB / 25-65K 公路) → 避开参数红海
  5. 情绪价值：加 UCI + 静电涂装 + 夜光烤漆 3 项品牌资产 → 跳出同质化
  6. 价格锚点：主战场 25-40K 锚定 Rockrider Core Lanster 27.7K / 锚点 45-55K 对标 Rockrider Pro Gen 4 45K
